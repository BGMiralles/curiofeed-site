curiofeed uses YouTube API Services

Privacy Policy

Last updated: 19 July 2026

Summary. curiofeed is a personal, single-operator application. It has no users other than its operator, collects no personal data from anyone, and uses YouTube API Services solely to upload videos to its operator's own YouTube channel.

1. Who operates curiofeed

curiofeed is a personal application built and operated by a single individual, who owns the YouTube channel it publishes to. It is not offered to the public, is not distributed or sold, and has no users other than its operator.

2. YouTube API Services

curiofeed uses YouTube API Services. By using curiofeed, its operator is agreeing to be bound by the YouTube Terms of Service.

Information handled through Google APIs is governed by the Google Privacy Policy, and curiofeed's use of that information adheres to the Google API Services User Data Policy, including its Limited Use requirements.

3. What data curiofeed accesses

  • The operator's own YouTube account. curiofeed authenticates with a single OAuth scope, https://www.googleapis.com/auth/youtube.upload, used exclusively to upload videos to the operator's own channel. It cannot and does not read, modify or delete data belonging to any other account or channel.
  • Public YouTube data. curiofeed reads public trend signals (search results and public video statistics) through the YouTube Data API, and uses them only to decide which topics to cover.

4. What curiofeed does not do

  • It does not collect, store, process or share personal data from any other person.
  • It has no user accounts, no sign-up, no analytics and no third-party tracking.
  • It does not sell, redistribute or publish YouTube API data.
  • It does not use YouTube API data for advertising, profiling or training models.
  • It does not share any data with third parties.

5. Storage, retention and deletion

Public trend data is kept only as a small local working set on the operator's own machine for topic selection, and is overwritten by subsequent runs. Operational records (selected topics, generated scripts, upload results) are stored locally and can be deleted by the operator at any time by removing the local database file.

Because curiofeed collects no data from any other person, there is no third-party data to retain, export or delete. Should anyone believe otherwise, they may contact the address in section 8 and any such data will be deleted.

6. Credentials and revoking access

OAuth credentials and tokens are stored locally on the operator's machine, excluded from version control, and are never transmitted to any third party.

The operator can revoke curiofeed's access to their Google account at any time through the Google security settings page at https://myaccount.google.com/connections (also reachable at https://security.google.com/settings/security/permissions). Revoking access immediately stops all further API use.

7. Changes to this policy

If this policy changes, the updated version will be published on this page with a new date.
