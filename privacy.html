Privacy Policy

curiofeed — last updated 19 July 2026

Who operates curiofeed

curiofeed is a personal, single-operator application. It is not offered to the public and has no users other than its operator, who owns the YouTube channel it publishes to.

Data collected from users: none

curiofeed has no user accounts, no sign-up, no analytics and no third-party tracking. It does not collect, store, process or share personal data from anyone. This website is a static description of the application and sets no cookies.

Google and YouTube data

curiofeed authenticates with the operator's own Google account using a single OAuth scope, https://www.googleapis.com/auth/youtube.upload, used exclusively to upload videos to the operator's own YouTube channel. It does not read, modify or delete data belonging to any other account or channel.

It also reads public YouTube trend data through the YouTube Data API, and uses it only to decide which topics to cover. That data is kept as a small local working set, is never redistributed or sold, and is never used for advertising, profiling or training models.

Use of information received from Google APIs adheres to the Google API Services User Data Policy, including its Limited Use requirements.

Credentials

OAuth credentials and tokens are stored locally on the operator's own machine, excluded from version control, and are never transmitted to any third party. Access can be revoked at any time from the operator's Google Account connections page.

Data retention and deletion

Because no data is collected from other people, there is nothing to retain or delete on their behalf. Local operational data (selected topics, generated scripts, upload records) lives only on the operator's machine and can be deleted by the operator at any time.

Changes

If this policy changes, the updated version will be published on this page with a new date.
